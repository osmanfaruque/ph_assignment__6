Frequently Asked Questions

1. The difference between var, let, and const?

var: Function-scoped variable declaration that can be redeclared and updated. Variables declared with var are hoisted to the top of their scope.

let: Block-scoped variable declaration that can be updated but not redeclared within the same scope. Not hoisted in the same way as var.

const: Block-scoped variable declaration for constants that cannot be updated or redeclared. Must be initialized at declaration time and maintains constant reference.

2. The difference between map(), forEach(), and filter()?

map(): Creates a new array by transforming each element in the original array. Returns the new array.

forEach(): Executes a function on each element of an array but doesn't return anything. Used for side effects.

filter(): Creates a new array containing only elements that pass a test function. Returns the filtered array.

3. Explain arrow functions and how they are different from regular functions?

Arrow functions are a more concise syntax introduced in ES6. Key differences from regular functions:

1. Shorter syntax with implicit returns for one-line functions

2. Don't have their own 'this' binding - they inherit 'this' from the parent scope

3. Cannot be used as constructors (no 'new' keyword)

4. Don't have arguments object or their own prototype

4. How JavaScript Promises work?

Promises are objects representing the eventual completion or failure of an asynchronous operation. They have three states:

Pending: Initial state, neither fulfilled nor rejected

Fulfilled: Operation completed successfully

Rejected: Operation failed

Promises use .then() for success handling, .catch() for error handling, and can be chained to avoid callback hell. They can also be combined with async/await for more readable asynchronous code.

5. How closures work in JavaScript?

A closure is a function that has access to its own scope, the outer function's variables, and global variables even after the outer function has finished executing.

Closures are created when a function is defined within another function, allowing the inner function to access the outer function's variables and parameters.

Common uses include data privacy (creating private variables), function factories, and maintaining state in event handlers or callbacks.
